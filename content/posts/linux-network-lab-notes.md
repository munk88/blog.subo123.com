写网络实验笔记时，我会优先记录四类信息。

## 1. 当前拓扑

先写清楚设备关系、IP 地址、网关、DNS 和关键服务。

```text
主路由: 192.168.88.1
实验主机: 192.168.88.41
旁路由: 192.168.88.15
```

这样以后回看时，不需要重新猜当时的环境。

## 2. 具体故障现象

不要只写“网络不通”，而是写清楚哪个客户端、哪个应用、访问哪个目标失败。

例如：

- 手机可以打开国内网站，但是 YouTube 加载失败
- Debian 可以 ping 网关，但是 DNS 解析超时
- 浏览器报错，命令行访问正常

## 3. 排查命令和结果

命令和结果要放在一起，方便以后复盘。

```bash
ip route
resolvectl status
curl -I https://www.google.com
```

## 4. 最终结论

最后写清楚根因、修复动作和验证结果。真正有价值的不是命令列表，而是判断链路。
